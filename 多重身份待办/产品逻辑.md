# 多重身份待办 — 产品逻辑基础文档

## 一、目标用户画像

**一句话：** 对现状不满、有改变欲望、但被"不知道从哪里开始"卡住的人。

**核心状态：**
```
感知：生活一团糟 / 别人的生活比我好
情绪：焦虑 + 羡慕 + 对自己失望
行动：想动 → 被选择压垮 → 放弃 → 更自责
```

**关键洞察：**
- 用过 Notion、打卡 App 等工具，但无法坚持
- 真正的障碍不是工具，是"设定目标本身需要成本"——用户不知道自己真正想要什么
- 触发点：社交媒体比较 + 长期积压的心理压力 + 人生节点

---

## 二、核心概念定义

### 身份（Identity）
- **定义：** 当下真实存在的角色，基于事实，不是愿景
- **特性：** 有时间性（离职后从"产品经理"变成"自由工作者"）；在某时间段内确定
- **规则：** 不被质疑，不限制数量（5个、8个、10个都可以）
- **两种状态：**

| 状态 | 定义 | 管理方式 |
|------|------|---------|
| 主动身份 | 这个阶段有目标、有 OKR、有每日任务 | 设定 OKR + 关键结果 |
| 维持身份 | 存在但暂无主动目标，只保证不变差 | 设定最低标准（底线） |

- **建议：** 同时主动建设的身份不超过 2-3 个（精力有限）

### 目标（Goal）
- **定义：** 用户在某身份下表述想实现的事
- **特性：** 不确定，不一定能反映真实想法，需要被质疑和挖掘
- **规则：** 每个主动身份下 1-2 个真实目标

### 真实动机（True Motivation）
- **定义：** AI 通过追问挖掘出的目标背后真正的渴望
- **逻辑举例：**
  ```
  用户说"学做饭"   → 真实动机：让父母觉得我独立了
  用户说"学画画"   → 真实动机：找到让自己平静下来的方式
  用户说"学英语"   → 真实动机：想要一个能出走的退路
  ```
- **重要性：** OKR 的 O 必须基于真实动机，否则再漂亮的 OKR 也在解决错误的问题

### 最低标准（Minimum Standard）
- **定义：** 维持身份的底线行为，不是目标，是"不让这个身份劣化"的最低保证
- **示例：**
  ```
  女儿    → 每周联系父母一次
  身体    → 不连续熬夜超过12点
  伴侣    → 每周有一次认真的对话
  ```

### 身份调节器（Identity Regulator）
- **定义：** 所有身份之间的相对精力权重，决定这个阶段每个身份获得多少时间资源
- **规则：** 相对权重，所有身份加起来 = 100%；调高一个，其余自动压缩
- **与主动/维持的关系：**

| 状态 | 调节器位置 | 含义 |
|------|-----------|------|
| 维持身份 | 锁定在最低权重 | 只保证底线任务，不参与主动分配 |
| 主动身份 | 可自由调节 | 权重越高，AI 生成的任务越多、目标推进越快 |

- **调整时机：**
  - 入门时：设定第一版权重
  - 每周复盘时：回顾上周分配是否符合预期，重新拨动
  - 生活阶段变化时：如换工作、备考期间，临时调整

- **示例：**
  ```
  平常状态：
    产品经理 ████████ 40%
    健身者   █████    25%
    写作者   █████    25%
    女儿     ██       10%（维持，锁定）

  备考/冲刺期：
    产品经理 ████████████ 60%
    健身者   ███          15%
    写作者   ███          15%
    女儿     ██           10%（维持，锁定）
  ```

## 三、核心逻辑链

```
身份（事实，不限数量）
  ↓
区分主动 / 维持
  ├── 维持身份 → 设定最低标准 → 调节器锁定最低权重 → 纳入每日计划（底线任务）
  └── 主动身份 → 身份调节器（相对权重分配）
                  ↓
                  目标（用户表述，不确定）
                    ↓ AI 质疑与挖掘
                  真实动机（OKR 的 O）
                    ↓
                  关键结果 KR（可衡量的行动）
                    ↓
                  每日任务（按权重比例分配时间）
```

## 四、用户旅程

### 阶段 0：触发

用户到来时处于情绪状态，不是理性状态。

```
触发来源：
  · 刷到朋友圈，看到别人的生活
  · 某个人生节点（生日、换工作、失恋）
  · 长期积压的不满终于溢出来

用户状态：焦虑、羡慕、想改变，但不知从哪里开始
```

产品在这一刻的任务：**立刻传递"我理解你"**，不是表单，不是问卷。

---

### 阶段 1：入门深度对话（一次性，产品最核心的体验）

#### 1.1 开场

不从"你想要什么"开始，从情绪和现状开始——因为用户情绪先于逻辑。

> AI：「最近怎么了，有什么让你觉得想改变的事吗？」

用户可以说任何东西：一团糟、看到朋友买房了、觉得自己每天都在浪费时间……AI 先接住情绪，再往下走。

#### 1.2 身份梳理

从用户说的内容里提取身份，或直接询问。

> AI：「我听到你提到了工作和家里的事。你现在生活里都扮演哪些角色？
>      工作上、家庭里、还有对自己来说——说说你现在是谁。」

**原则：**
- 用户说多少算多少，不打断，不限制
- AI 不预设身份分类（不说"你有没有职业身份、家庭身份"这种）
- 用户说完，AI 整理复述，确认有没有遗漏

```
示例输出：
「我听下来，你现在有这几个身份：
  · 产品经理（工作）
  · 女儿（家庭）
  · 在健身但不稳定的人（自己）
  · 想写作但一直没开始的人

有没有漏掉的？」
```

#### 1.3 主动 / 维持分类

> AI：「这些身份里，现在哪些是你想主动改变或发展的？
>      哪些只要维持现状、不让它变差就够了？」

用户做选择，AI 不判断对错。如果用户选了超过 3 个主动身份，AI 轻轻提示：

> 「这几个都很重要。不过同时推进太多，精力容易分散。
>   你现在最想先动起来的是哪 2-3 个？其他的我们先设底线就好。」

#### 1.4 维持身份设底线

针对每个维持身份，AI 问：

> 「作为[身份]，你觉得最低限度应该做到什么，才不会觉得自己在这件事上退步了？」

底线要具体、可执行，不是目标。

```
示例：
女儿    → 每周打一次电话
身体    → 每天走路不少于 20 分钟
```

#### 1.5 主动身份挖目标

针对每个主动身份，AI 问：

> 「作为[身份]，你现在最想实现什么？
>   不用想太多，说第一个冒出来的。」

**用户说出表层目标后，AI 开始追问：**

追问的方式像朋友，不像医生。轻盈，有好奇心，不评判。

```
追问层次一（为什么）：
「你为什么想[stated goal]？」

追问层次二（如果实现了）：
「如果[stated goal]实现了，你的生活会有什么不同？」

追问层次三（最在意的）：
「在这件事里，你最在意的其实是什么？」

AI 反射真实动机：
「我听下来，好像你真正想要的是[true motivation]，
 [stated goal] 是你以为能得到它的方式。
 这个理解对吗？」
```

用户确认（或纠正）后，真实动机确立。

#### 1.6 OKR 结构化

基于真实动机，AI 生成第一版 OKR。

**呈现方式：** 对话式，不是表格——避免用户感觉在填写企业文件。

```
「好，基于我们聊的，我帮你整理了一下这个季度的方向：

 身份：在健身但不稳定的人
 你真正想要的：在朋友面前有身体自信，不再因为体型感到回避

 这个季度的目标（O）：
   成为一个对自己身体感到自豪的人

 关键结果（KR）：
   · 每周完成 3 次 30 分钟以上的运动
   · 3个月内体脂减少 3%
   · 能在朋友聚会时穿自己回避的那件衣服出门

 你觉得这样说准确吗？有什么想调整的？」
```

用户可以逐条修改，AI 实时更新。

#### 1.7 设定身份调节器（初始权重）

所有身份的 OKR 和底线都确认后，AI 引导用户设定第一版权重。

> 「现在你有了这几个主动身份，在接下来这段时间里，
>   你希望把更多精力放在哪里？帮我拨一下这个调节器。」

呈现方式：可视化的相对权重分配（如滑块组或饼图），实时显示百分比，
所有主动身份加起来 = 100%（维持身份固定在最低权重，不参与分配）。

用户调整后，AI 确认：

> 「好，接下来 AI 安排每日任务时，会按照这个比例给你分配精力。
>   如果哪天觉得不对，可以随时调整。」

#### 1.8 确认进入日常模式

> 「好了，你现在有了清晰的方向，也知道这阶段最想投入的是哪里。
>   从明天开始，我每天早上帮你安排今天该做什么。」

---

### 阶段 1.9：随时添加新身份（贯穿全生命周期）

身份不是一次性设置完的，用户的生活会变化，产品需要跟着生长。

**触发场景：**

```
用户主动：「我最近开始学吉他了，想把它加进来」→ 点击"添加身份"

AI 发现线索：用户日复盘提到「今天去上了陶艺课，感觉很好」
  AI：「你好像开始了一个新的事情，要把它加进来管理吗？」

生活变化：用户换工作 / 结婚 / 生孩子
  AI：「你的身份有变化，我们聊聊新的方向吗？」
```

**对话流程：** 与入门 1.2-1.6 步骤完全一致，范围缩小到这一个身份。

```
Step 1：确认身份名称
Step 2：主动 or 维持？
Step 3a（维持）：设最低标准
Step 3b（主动）：挖目标 → AI 质疑 → 真实动机 → OKR
Step 4：确认后自动合并进每日计划
```

**预计时长：** 单个新身份 8-12 分钟。

**连带好处：** 入门时不必把所有身份都挖完——先从 1-2 个最想改变的开始（15-20 分钟），其他身份随时可以添加。产品是生长的，不是一次性配置完的。

### 阶段 2：日常执行

#### 2.1 每日计划生成

每天早晨，AI 根据以下信息生成当日计划：
- 各主动身份的 KR 进度
- 维持身份的底线任务
- 用户的精力习惯（早鸟 / 夜猫子）
- 固定日程（会议、接送等）

**时间边界原则（排任务前先划边界）：**

AI 在排任务前，先将一天的时间划分为四类，只在「可调度时间」内排任务：

| 时间类型 | 归属 | AI 处理方式 |
|----------|------|------------|
| 机能时间（睡眠、三餐、午休） | 身体基本需求 | 硬性保留，不排任何任务 |
| 工作时间（固定上班时段） | 职业身份 | 只可排跨身份的工作任务，私人目标不介入 |
| 私人时间 | 不属于任何身份 | AI 保护并可动态调整，不排任务 |
| 可调度时间 | 各身份目标 | AI 按调节器权重分配任务 |

**私人时间的调整逻辑：**

私人时间不是固定的，AI 会根据当天情况动态调整：
- 设定一个用户认可的**最低保障时长**（默认 1.5 小时，用户可修改）
- 当天任务密度高时，AI 主动压缩目标任务，而非继续占用私人时间
- 长期通过用户复盘反馈（「今天太累」/「今天状态不错」）学习这个人的节奏，校准私人时间的合适量

> 核心逻辑：私人时间是「不被目标占据的结果」，不是被规划进来的任务。AI 的职责是保护它，而不是填满它。

**计划的呈现原则：**

每个任务不只显示"做什么"，还显示"为什么做"——连接到真实动机。

```
示例：
09:00  运动 30 分钟
       · 健身者身份 · 「成为对自己身体自豪的人」

12:30  给妈妈发条消息问问近况
       · 女儿身份 · 底线任务

20:00  写 300 字关于今天工作的思考
       · 想写作的人 · 「找到自己的表达声音」
```

#### 2.2 执行中的交互

- **完成任务：** 勾选，可加备注（做了什么、感受如何）
- **跳过任务：** 标记跳过，可以说明原因（AI 后续会注意这个模式）
- **临时加任务：** 可手动添加，AI 帮助归类到对应身份

#### 2.3 日内提醒（轻量）

不做频繁推送，只在关键时间点提醒：
- 早上：今日计划就绪
- 晚上：今日复盘时间

---

### 阶段 3：复盘（进化机制）

复盘是目标持续进化的核心机制，分三个层次。

#### 3.1 日复盘（每晚，5 分钟内）

轻量，不是表单，是对话。

> AI：「今天怎么样？」

用户自由回答，AI 根据当日计划追问一个具体问题：

> 「我看到运动这件事跳过了，发生什么了？」

AI 记录原因，不评判，只观察。

#### 3.2 周复盘（每周，15-20 分钟）

AI 主动发起，展示本周数据，提出洞察。

```
本周小结：

身份投入分布：
  · 产品经理    ████████ 60%
  · 健身者      ███ 20%
  · 想写作的人  ██ 15%
  · 底线任务    █ 5%

AI 观察：
  「这周运动完成率只有 33%，连续两天因为工作加班跳过。
   你之前说运动对你很重要——我们要重新看看这个 KR 的节奏吗？」

本周一个值得思考的问题：
  「如果现在的你见到三个月后的你，你最希望那个人有什么不同？」

身份调节器回顾：
  「上周你给健身分配了 25%，但实际完成率只有 10%。
   是权重设高了，还是这周特殊情况？下周要调整吗？」
```

#### 3.3 季度校准（每季度，或按需触发）

重新审视 OKR 是否还有意义，可能触发新一轮目标挖掘。

**AI 触发校准的条件：**

| 触发条件 | AI 的行动 |
|---------|---------|
| 某任务连续 2 周被跳过 | 「我注意到你一直在回避这件事，我们聊聊？」 |
| OKR 接近到期（剩最后 2 周） | 发起季度回顾对话 |
| 用户主动说「这个目标好像不重要了」 | 立即开启校准对话 |
| 某身份持续 0 任务（被遗忘） | 「[身份]这边最近完全没有动作，还在你的计划里吗？」 |

**校准对话流程：**

```
1. AI 展示这个 OKR 的完成情况和数据
2. 开放式问题：「这个目标现在对你还重要吗？」
3. 如果重要但没做：重新挖掘阻力（是目标定错了？还是 KR 不对？）
4. 如果不再重要：确认是否移除，是否有新方向冒出来
5. 更新 OKR，进入新一个周期
```

---

### 整体弧线

```
迷茫的人（不知道自己想要什么）
    ↓ 入门对话（挖掘真实动机，梳理身份）
有方向的人（知道自己是谁，这阶段在做什么）
    ↓ 日常执行（把方向变成今天的行动）
在行动的人（每天知道为什么做这件事）
    ↓ 复盘（看见模式，调整方向）
    ↓ 校准（目标随成长进化）
活得像自己想要的样子的人
```

## 五、产品定位差异

| 现有工具 | 这个产品 |
|---------|---------|
| 假设用户已知目标，帮他追踪 | 帮用户发现真实目标，再追踪 |
| 「告诉我你的 OKR」 | 「我们一起找到你真正想要的」 |
| 目标 = 用户填写的内容 | 目标 = AI 挖掘出的真实动机 |
| 身份是标签 | 身份是有事实基础的角色，有主动/维持之分 |
| 限制目标数量 | 身份不限，主动身份建议 2-3 个 |
