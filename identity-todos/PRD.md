# 身份待办（Identity Todos）— 产品方向

## 一句话

一个 AI 驱动的身份管理 + 待办规划工具，让用户在多重人生角色之间无缝切换，不会遗漏任何一个身份的责任。

## 目标用户画像

**姓名：** 林 / 姐姐
**年龄：** 30-40 岁，职场中坚
**身份标签：** 产品设计师 · 独身者 · 旅游爱好者 · 独生子女
**核心痛点：**
- 不同领域的事务混在同一个 todo list 里，大脑切换成本高
- 没有一个系统能理解她的"全貌"——工作日程、父母体检、旅行签证、日常买菜
- 明明有很多事要做，但优先级不清晰，容易陷入"什么都想做，什么都没做好"
- 独身意味着所有决策和事务都要自己扛，没人帮忙兜底

## 核心概念：身份（Identity）

每个用户有一个或多个「身份」。每个身份有：
- **名称**（产品设计师、独身者、旅游爱好者）
- **emoji / 图标**
- **关联事务列表**
- **AI 上下文**（系统知道在这个身份下，用户关注什么）

示例：
```
🏢 产品设计师
  - 今天下午 3 点评审 PRD
  - 周五前给业务方回方案
  - 关注：竞品分析更新、设计系统迭代

🏠 独身者
  - 今晚做晚饭
  - 交水电费
  - 关注：睡眠质量、房间整洁度

✈️ 旅游爱好者
  - 办签证（截止 6/15）
  - 订机票
  - 关注：特价通知、攻略收藏

👨‍👩‍👧 独生子女
  - 带妈妈体检（预约中）
  - 关注：家庭群消息、慢病管理
```

## AI 核心能力

1. **全量资料接入** — 日历、邮件、笔记、聊天记录、相册（旅行照片）、健康数据
2. **按身份拆分** — 自动识别一条新事务属于哪个身份
3. **智能优先级** — 根据截止时间、重要性、身份活跃度排序
4. **主动提醒** — "姐姐，你上周说这个月要带妈妈体检，现在该约号了。"
5. **每日简报** — 每天早上按身份列出今天要做的几件事

## 交互模式

- **默认视图：身份仪表盘** — 每个身份一张卡片，卡片上显示该身份当前最重要的 3 件事
- **切换身份** — 点击身份的 tab/card，进入该身份的事务列表
- **快速录入** — 用自然语言增加事务："下周三下午带妈妈去医院复查"
- **回顾** — 每天/每周回顾，看看哪个身份被忽略了

## 差异化定位

| 同类产品 | 我们的不同 |
|---------|-----------|
| Todoist / TickTick | 按项目分组，但不理解"你这个人" |
| Notion / 飞书 | 灵活但需要自己搭，AI 介入浅 |
| 各种 AI 助手 | 都是对话式，没有"身份"的结构化输出 |
| 日历（Google/Apple） | 只看时间，不看人生 |

**独特价值：** 不只是给你一个清单，而是帮你理解——你现在的优先级是哪个身份。

## MVP 功能清单

1. **身份管理**：增删改身份名称/图标/描述
2. **事务管理**：CRUD 事务，每个事务属于一个身份
3. **自然语言录入**：一句话添加事务，AI 自动分类到身份
4. **AI 每日规划**：每天早上推送今天各身份的重点
5. **Web 版**：PWA / 响应式，手机电脑都能用
6. **日历导入**：读取系统日历自动归类

## 技术方向

- **前端：** React / Next.js（Web-first 方便跨平台）
- **AI 层：** 本地 Agent（也就是我）先做 POC，后续可独立
- **数据存储：** 用户可选本地存储或同步
- **日历集成：** iCal / Google Calendar API

## 后续可能

- 多个身份之间的冲突检测（工作和家庭事务撞车）
- 身份雷达图：可视化各身份的关注度是否平衡
- "身份切换模式"：下班自动切到独身者模式
- 共享身份：和父母的健康事务共享
