# 身份待办（Identity Todos）— 产品设计文档 v2

## 一句话

一个 AI 驱动的身份管理系统，将你的多重人生角色拆解为日程 + 待办，自动化解模糊目标，智能协调冲突。

---

## 用户故事

姐姐每天打开 App，看到：

```
🗓️ 今天 5月6日 周三

  08:00 ── 🌅 起床（🏠 独身者）
  09:00 ── 📝 PRD 评审 @ 飞书（🏢 产品设计师）
  10:30 ── 💬 业务对齐会（🏢 产品设计师）
  12:00 ── 🍜 午饭（🏠 独身者）
  14:00 ── 💧 带妈妈去社区医院拿药（👨‍👩‍👧 独生子女）
  15:00 ── ✈️ 看国庆北海道攻略（✈️ 旅游爱好者）
  19:00 ── 🏃 跑步 30 分钟（🏠 独身者）
```

可以在右上角筛选「只看某个身份」，比如：

```
[全部] [🏢设计师] [🏠独身者] [✈️旅游] [👨‍👩‍👧子女]
```

点击某身份 → 日程变灰，只高亮该身份的事务。

## 待办系统：清晰与模糊双轨

### 清晰待办 ✅
- 时间、地点、人明确的
- "下周三下午3点协和医院复查"
- 直接放上时间轴

### 模糊目标 🌫️ → AI 拆解
- "7月英语突飞猛进"
- "今年学会自由泳"
- "年底带爸妈去一次旅行"
- AI 自动拆成可执行子任务 + 推荐排期
- 用户确认/修改后落上时间轴

**拆解交互示例：**

```
姐姐，你说「7月英语突飞猛进」，我来帮你拆一下👇

┌────────────────────────────────┐
│ 📅 每周一/三/五 08:00 听力30min │
│ 📅 每天通勤 单词20个            │
│ 📅 每周六 14:00 外教口语1h      │
│ 📅 7/28 雅思模考                │
│                                │
│ ✅ 确认排期   ✏️ 修改   ❌ 删除  │
└────────────────────────────────┘
```

## 浮动优先级系统 ⚖️

### 核心逻辑

不是「用户设置优先级」，而是 **系统从用户行为中推断优先级**。

| 触发条件 | 系统行为 |
|---------|---------|
| 两个目标排期冲突 | 弹窗问用户选哪个 |
| 用户连续 3 次选择 A 而非 B | A 优先级上升，B 下降（下次冲突自动倾向 A） |
| 某个目标 2 周没被执行 | 优先级逐渐衰减 |
| 用户手动调整时间 | 系统记录偏好 |

### 优先级是「浮动的」

- 不是静态的 P0/P1/P2
- 每天起床，系统根据 **当前语境** 重新排序
  - 今天是工作日 → 🏢 设计师事务排前
  - 今天是周六 → ✈️🌲 旅游/生活排前
  - 妈妈昨天说身体不舒服 → 👨‍👩‍👧 子女事务权重自动拉升

## 信息架构

```
├── 🏠 首页（日程时间轴）
│   ├── 按身份筛选栏 [全部｜🏢｜🏠｜✈️｜👨‍👩‍👧]
│   ├── 今日进度条（已完成/总量）
│   ├── 身份分布图（时间占比色块）
│   └── 底部浮动输入框「➕ 添加待办」
│
├── 📋 待办列表
│   ├── 清晰待办（已排期）
│   ├── 模糊目标（待拆解）
│   └── 已完成（归档）
│
├── 👤 身份管理
│   ├── 各身份的详情
│   ├── 该身份下的待办汇总
│   └── AI 对该身份的上下文理解
│
├── 📊 周报 / 月报
│   ├── 各身份时间投入占比
│   ├── 目标完成率
│   └── 优先级变化趋势
│
└── ⚙️ 设置
    ├── 日历接入
    ├── AI 资料源管理
    └── 通知偏好
```
